\## 2. Estrategia de Git

\- Rama principal: `main` → siempre estable, lista para producción.

\- Rama de desarrollo: `develop` → integración de funcionalidades.

\- Ramas de características: `feature/nombre` → desarrollo de nuevas funcionalidades.

\- Ramas de corrección: `hotfix/nombre` → arreglos urgentes.

\- Flujo:

&nbsp; 1. Crear rama `feature` desde `develop`.

&nbsp; 2. Hacer commits en la rama feature.

&nbsp; 3. Merge a `develop` cuando la funcionalidad esté lista.

&nbsp; 4. Merge de `develop` a `main` para producción.



\## 3. Creación del proyecto

\- Lenguaje: JavaScript (Node.js)

\- Framework: Express.js

\- Archivo principal: `index.js`

\- Servidor corriendo en `http://localhost:3000`

\- Código subido a GitHub en la rama `main`.
